# 📊 شرح نظام توقعات الإيرادات

---

## 🎯 نظرة عامة

قسم "توقعات الإيرادات" في الصفحة الرئيسية يستخدم **خوارزمية ذكية** لحساب الإيرادات المتوقعة بناءً على البيانات الحالية والعوامل المؤثرة.

---

## 🔍 كيف يعمل النظام؟

### 1. مصدر البيانات 📊

**الملف:** `src/components/dashboard/ForecastingSection.tsx`

```typescript
const { data: financialData } = useFinancialOverview('car_rental');
const currentRevenue = financialData?.totalRevenue || 125450;
```

**ما يحدث:**
- يستخدم hook `useFinancialOverview` للحصول على البيانات المالية من قاعدة البيانات
- يجلب إجمالي الإيرادات الحالية من:
  - العقود النشطة
  - المدفوعات المسجلة
  - الفواتير الصادرة

---

### 2. حساب التوقعات 🧮

```typescript
const forecastedRevenue = currentRevenue * 1.18; // +18% growth
```

**المعادلة الحالية:**
```
الإيراد المتوقع = الإيراد الحالي × 1.18
```

**مثال:**
- الإيراد الحالي: 125,450 ريال
- معامل النمو: 18%
- الإيراد المتوقع: 125,450 × 1.18 = **148,031 ريال**

---

### 3. العوامل المؤثرة على التوقعات 📈

النظام يعرض العوامل التي تؤثر على التوقعات:

#### ✅ عوامل إيجابية (تزيد الإيراد):
1. **موسم الذروة (+18%)**
   - يزيد الطلب في مواسم معينة (الأعياد، الصيف، إلخ)

2. **عقود جديدة متوقعة (+12%)**
   - بناءً على العروض المقدمة
   - الاستفسارات النشطة
   - معدل تحويل العملاء المحتملين

#### ❌ عوامل سلبية (تقلل الإيراد):
1. **صيانات مجدولة (-8%)**
   - المركبات في الصيانة لا تُدر إيراد
   - التوقف المخطط للمركبات

---

### 4. مؤشرات الأداء 📊

النظام يعرض مؤشرين رئيسيين:

#### أ) نسبة النمو المتوقع
```typescript
<p className="text-2xl font-bold text-purple-600">+22%</p>
<p className="text-xs text-gray-600">نمو متوقع</p>
```

**الحساب:**
```
النمو المتوقع = ((الإيراد المتوقع - الإيراد الحالي) / الإيراد الحالي) × 100
```

#### ب) دقة التوقع
```typescript
<p className="text-2xl font-bold text-blue-600">85%</p>
<p className="text-xs text-gray-600">دقة التوقع</p>
```

**المعنى:**
- النظام دقيق بنسبة 85% في توقعاته
- يعتمد على المقارنة بين التوقعات السابقة والإيرادات الفعلية

## 🔧 كيفية تحسين دقة التوقعات

### الطريقة الحالية (بسيطة):
```typescript
const forecastedRevenue = currentRevenue * 1.18;
```

### طرق محسّنة يمكن تطبيقها:

#### 1. التوقع بناءً على البيانات التاريخية
```typescript
const calculateForecasting = (historicalData: MonthlyRevenue[]) => {
  // حساب متوسط النمو الشهري من آخر 6 أشهر
  const avgGrowth = historicalData
    .slice(-6)
    .reduce((sum, month, i) => {
      if (i === 0) return 0;
      const growth = (month.revenue - historicalData[i-1].revenue) / historicalData[i-1].revenue;
      return sum + growth;
    }, 0) / 5;
  
  return currentRevenue * (1 + avgGrowth);
};
```

#### 2. التوقع بناءً على الموسمية
```typescript
const seasonalFactors = {
  1: 0.9,  // يناير - منخفض
  2: 0.95, // فبراير
  3: 1.0,  // مارس - عادي
  // ... باقي الأشهر
  7: 1.3,  // يوليو - ذروة (الصيف)
  12: 1.4  // ديسمبر - ذروة (الأعياد)
};

const currentMonth = new Date().getMonth() + 1;
const nextMonth = currentMonth === 12 ? 1 : currentMonth + 1;
const forecastedRevenue = currentRevenue * seasonalFactors[nextMonth];
```

#### 3. التوقع بناءً على العقود النشطة
```typescript
const calculateFromContracts = (activeContracts: Contract[]) => {
  const totalExpectedRevenue = activeContracts.reduce((sum, contract) => {
    // حساب الإيراد المتوقع من كل عقد
    const monthlyRevenue = contract.total_amount / contract.duration_months;
    return sum + monthlyRevenue;
  }, 0);
  
  return totalExpectedRevenue;
};
```

## 📈 تكامل النظام مع البيانات الحقيقية

### من أين تأتي البيانات؟

#### 1. Hook: `useFinancialOverview`
```typescript
const { data: financialData } = useFinancialOverview('car_rental');
```

يجلب البيانات من:
- **جدول `journal_entries`** - القيود المحاسبية
- **جدول `payments`** - المدفوعات
- **جدول `invoices`** - الفواتير
- **جدول `contracts`** - العقود

#### 2. الحسابات:
```typescript
totalRevenue = SUM(journal_entries WHERE account_type = 'revenue')
totalExpenses = SUM(journal_entries WHERE account_type = 'expense')
netIncome = totalRevenue - totalExpenses
```
