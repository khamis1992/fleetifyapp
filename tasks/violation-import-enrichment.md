# تحسين نظام استيراد المخالفات - إكمال البيانات الناقصة

## الهدف
عند رفع ملف PDF للمخالفات، يقوم النظام بـ:
1. البحث عن المخالفات الموجودة مسبقاً في قاعدة البيانات
2. مقارنة البيانات المستخرجة من PDF مع البيانات الموجودة
3. تحديد الحقول الناقصة في قاعدة البيانات
4. إتاحة خيار تحديث/إكمال البيانات الناقصة من PDF

## الحقول القابلة للإكمال
- [x] location (الموقع)
- [x] violation_description (وصف المخالفة)
- [x] violation_time (وقت المخالفة)
- [x] issuing_authority (الجهة المصدرة)
- [x] contract_id (ربط العقد)

## المهام
- [x] 1. إضافة دالة للبحث عن المخالفات الموجودة بناءً على رقم المخالفة أو المرجع ✅
- [x] 2. إضافة منطق مقارنة البيانات الناقصة ✅
- [x] 3. إضافة تبويب جديد "إكمال البيانات" في واجهة الاستيراد ✅
- [x] 4. إضافة دالة لتحديث المخالفات الموجودة بالبيانات الناقصة ✅

## ملخص التغييرات
تم تحديث ملف `useViolationMatching.ts` بإضافة:
1. `useViolationEnrichment` hook جديد للبحث عن المخالفات الموجودة ومقارنة البيانات
2. `findExistingViolations` - للبحث عن المخالفات بناءً على أرقام المرجع
3. `compareAndFindMissing` - لمقارنة البيانات وإيجاد الحقول الناقصة
4. `enrichViolations` - لتحديث المخالفات الموجودة بالبيانات الجديدة

تم تحديث ملف `TrafficViolationPDFImport.tsx` بإضافة:
1. تبويب جديد "إكمال البيانات" (Enrich Data)
2. عرض المخالفات التي لديها بيانات ناقصة يمكن إكمالها
3. زر لتحديث البيانات الناقصة من PDF
